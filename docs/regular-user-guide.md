# Regular User Guide

This guide is for Simply Plural users who want to preserve their data locally.

PluralBridge is currently an early preservation toolkit. Some steps still require command-line tools, but this guide is intended to become the plain-language path for users who know how to install and run software.

## Current Goal

The current goal is to help users preserve their Simply Plural data before it becomes inaccessible.

The preservation process is intended to save:

- account/system data
- member data
- fronting history
- custom fields
- privacy buckets
- friends data
- notes, where available
- avatar images, where available

## Privacy Warning

Your exported data may contain deeply private information.

Do not post exported files publicly. Do not upload them into issues, pull requests, screenshots, chat rooms, or shared folders unless you have intentionally redacted them.

Your API token is private. Treat it like a password.

## Current Status

The first version of PluralBridge is developer-oriented.

Regular-user documentation will be expanded as the process becomes easier to run safely.

## Export First, Refine Later

Run a preservation export as soon as you can.

Do not wait until your Simply Plural account is perfectly cleaned up, reorganized, renamed, edited, or updated before making a backup.

PluralBridge exports are safe to run multiple times while the Simply Plural service remains available. If you make changes in Simply Plural after one export, run PluralBridge again and create a newer export.

This matters because the most important preservation step is getting a copy of your data while export is still possible. A rough export made today is safer than a perfect export that never happens.

Recommended workflow:

1. Run an export now.
2. Keep that export private and backed up.
3. Make any Simply Plural edits or cleanup you want.
4. Run another export.
5. Keep the newest export, and optionally keep older exports as dated snapshots.

PluralBridge should be treated as a repeatable preservation tool. The export process is designed so users can capture data first, then rerun later as their Simply Plural data changes.
