Task 26 groups schema design

Groups are system-scoped organization records.
Groups preserve source/import group structure.
Groups are data/import support.
Groups are not Accounts.
Groups are not Systems.
Groups are not Memberships.
Groups are not Roles.
Groups are not security boundaries.

Schema tables

dbo.pb_groups stores one row per group.
dbo.pb_group_members stores one row per member assignment to a group.

Group membership model

Group membership is many-to-many.
A member can belong to more than one group.
A group can contain more than one member.
Group membership is modeled through dbo.pb_group_members.
Group membership is not modeled as dbo.pb_members.GroupId.

System scope

Groups belong to one SystemId.
Group-member assignments belong to the same SystemId.
Groups do not replace System.
Groups do not replace Membership.

Runtime boundary

Task 26 adds schema only.
Task 26 does not activate UI behavior.
Task 26 does not activate runtime group behavior.
Task 26 does not change authorization behavior.
Task 26 does not change /api/me behavior.
