Task 27 child system relationship design

Child system relationships model hierarchy between Systems.
A System can have a parent System.
A System can have child Systems.
A child System remains a System.
A child System remains a protected data namespace.

Schema table

dbo.pb_system_relationships stores one row per System hierarchy relationship.
SystemId identifies the child or current System.
ParentSystemId identifies the parent System.
ParentSystemId is NULL for the root System row.
RelationshipRank stores hierarchy depth.

Root rule

The root System has ParentSystemId = NULL.
The root System has RelationshipRank = 1.
Child Systems have ParentSystemId set to their parent SystemId.
Child Systems have RelationshipRank greater than their parent.

Boundary rule

System relationships do not create Accounts.
System relationships do not create Memberships.
System relationships do not create Roles.
System relationships do not change authorization behavior in Task 27.
System relationships do not change /api/me behavior in Task 27.

Task 27 adds schema only.
Task 27 does not activate UI behavior.
Task 27 does not activate runtime child-system behavior.
