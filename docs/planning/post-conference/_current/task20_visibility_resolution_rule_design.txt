Rule name: PB first, SP fallback

Resolution order:
1. Check for a PB visibility scope assigned to the resource.
2. If PB visibility scope exists, use the PB visibility scope decision.
3. If PB visibility scope is absent, fall back to SP privacy bucket behavior.

Conflict rule:
When PB visibility scope and SP privacy bucket data both exist for the same resource, PB visibility scope wins.

Authority split:
PB visibility scopes are the PB runtime visibility authority.
SP privacy buckets remain Simply Plural import/export fidelity data.
